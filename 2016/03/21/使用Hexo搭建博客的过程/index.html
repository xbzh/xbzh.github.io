使用Hexo搭建博客的过程

发表于 2016-03-21   |  

工作后慢慢发现，人是应该经常总结，尤其是像我这样的普通人。便有了搭个博客的想法，于是便有了这个博客，在搭建过程中遇到了一些问题，简单记录一下:)

框架选择

Github pages
Github提供了pages服务，可以把自己的站点也托管到github上。这样就算你自己电脑坏了，你博客的内容也不会丢失。
Hexo
Hexo和Jekyll是快速搭建静态博客的利器，并且可以和github完美结合，一键部署，真的是很方便。二者之间我选择了用Hexo，其实不是因为Hexo更快，而是完全凭感觉，哈哈。

安装

Github
首先需要先注册github账号，废话，不然我们的pages放哪:)。Github Pages是username.github.io这种类型的站点，username为自己的github的用户名，所以登录github后，创建一个username.github.io的repo即可。
安装Hexo
Hexo是用nodejs实现的框架，安装Hexo前需要先安装Node.js。安装完Node.js后，利用npm命令来安装Hexo，在你要安装的文件夹下执行:

npm install -g hexo

安装完成后，执行初始化：

hexo init

现在准备工作已经都完成了，我们要生成页面了:)

hexo generate

启动本地服务：

hexo server

在浏览器中输入http://localhost:4000就可以看到效果了，嘿嘿。

部署到Github

修改_config.yml文件，找到deploy字段，添加要部署的repo。

type: git
repository: https://github.com/username/username.github.io.git
branch: master

注意在配置的时候冒号和值之间需要有空格，不然会出错。此时就可以一键部署了：

hexo deploy

额，好像有错误，如下：

ERROR Deployer not found: git

这是因为git的部署模块默认没有安装，安装hexo-deployer-git

npm install hexo-deployer-git —save

这时再执行deploy命令，就可以啦。当部署完成后，就可以访问username.github.io啦，嘿嘿。
发表和删除文章
发表文章直接执行：

hexo new ‘title’

这时就会生成一个markdown的文件，编辑该文件就可以了。删除文章的话，直接把这个markdown文件删除就可以了。发表和删除操作，都需要重新执行genetate和deploy，不然只在本地有效。

添加多说

多说系统是目前比较受欢迎的评论系统，其部署也很方便。首先需要到多说官方网站上注册一个账号，获得你的多说的shortname，指二级域名名称。然后在_config文件中添加你的shortname:

duoshuo_shortname: yourshortname

如果使用Next主题的话，此时多说系统就已经配置好了。如果你使用默认主题landscape的话，需要到landscape/layout/_partial/article.ejs中把多说的通用代码添加进去，具体操作可以参考：Hexo使用多说教程

Next主题

Next是一个极简主义风格的主题，我也灰常喜欢。安装Next主题非常简单，直接把该主题clone到本地theme文件夹中。

git clone https://github.com/iissnan/hexo-theme-next.git

创建tags、categories、about页面，默认这三个页面是没有生成的，以tags为例。

hexo new page tags

然后编辑/source/tags/index.md文件，添加tags属性。

type: “tags”

重新生成后，就可以看到首页多了标签页面。

遇到的问题

在执行hexo的命令的时候，可能会出现以下问题：

Error: Cannot find module ‘./build/Release/DTraceProviderBindings’

热心网友们给出的解决方法：

npm install hexo –no-optional

在执行deploy部署的时候，有时会出现卡住的现象，长时间没有反应，尤其是当部署较大的文件时。这时可以中断命令，将./deploy_git文件夹删除，再重新执行部署命令即可。
